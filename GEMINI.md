# tmxt - Project Overview

**tmxt** is a survivor-style game project focused on creating a satisfying, addictive, and polished gameplay experience. The project follows a structured 5-phase development roadmap designed to prioritize core "feel" and game loops before adding complexity.

## Project Roadmap

The development is divided into five distinct phases, as outlined in `Phases.txt`:

### Phase 1: Core Game Loop (Foundation)
- **Goal:** Create a playable, satisfying prototype.
- **Key Features:** Smooth player movement (WASD/Arrows), auto-shooting nearest enemies, basic enemy spawning from edges, collision detection, and a basic UI (HP/Timer).
- **Success Metric:** Movement feels smooth and killing enemies is satisfying.

### Phase 2: Progression System
- **Goal:** Transition from a prototype to an actual game.
- **Key Features:** XP Orbs with magnet effects, leveling system (modular upgrades), and a level-up screen with random choices.
- **Success Metric:** Each level makes the player feel noticeably stronger.

### Phase 3: Variety + Scaling
- **Goal:** Make the game dynamic and challenging.
- **Key Features:** Multiple enemy types (Fast, Tank, Spawner), difficulty scaling over time, and intensity spikes.
- **Success Metric:** Late-game feels chaotic and stressful in a rewarding way.

### Phase 4: Juice (The "Feel")
- **Goal:** Enhance the sensory experience.
- **Key Features:** Screen shake, hit flashes, particle bursts, sound design (shooting/hits/level-up), and visual feedback.
- **Success Metric:** Every action provides immediate and satisfying feedback.

### Phase 5: Depth (Optional/Expansion)
- **Goal:** Add long-term engagement.
- **Key Features:** Bosses, new weapons, meta-progression, and multiple characters.

## Development Principles

- **Polish First:** Do not add new features until the current phase feels "right." Phase 1 movement and combat must be "fun" before moving to Phase 2.
- **Modular Upgrades:** Keep the upgrade system flexible for easy expansion.
- **Feedback Driven:** Prioritize "Juice" (visual/audio feedback) as it is critical to the game's success.

## Key Files
- `Phases.txt`: The primary roadmap and design document for the project.

## Usage
This directory serves as the foundation for the **tmxt** game. Future interactions should refer to `Phases.txt` to determine the current development focus and ensure adherence to the project's phased approach.
